# Security Policy

## Supported Versions

Security fixes are provided on the `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for suspected security problems.

Instead, contact the maintainer privately with:

- a short summary
- impact assessment
- reproduction steps or proof of concept
- affected commit or release, if known

If private contact details are not available yet, open a minimal issue asking for a secure contact channel without disclosing the vulnerability details.

## Expectations

- Saved server passwords are optional and stored locally only.
- Exported favorites intentionally omit saved passwords.
- `omp-client.dll` must remain an explicit manual download and must not be committed to this repository.
